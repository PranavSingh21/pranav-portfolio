# Savvy — Roadmap V2

## Overview

Savvy V1 validates one core behavior:

Users are willing to track money conversationally if the friction is low enough.

V2 focuses on making Savvy smarter, more personalized, and more useful over time.

---

## 1. Persistent User Accounts

Move from local device memory to persistent user accounts.

This enables:
- multi-device continuity
- persistent spend history
- account recovery
- long-term usage

---

## 2. Category Learning Memory

Savvy should remember user corrections.

Example:
If a user classifies “Coke Zero” as Personal once, Savvy should learn that preference.

This reduces repeated clarification and improves personalization.

---

## 3. Monthly Trends & Insights

Move from static totals to behavioral insights.

Examples:
- “You spent 22% more on Eating Out this month.”
- “Transport spend is trending up.”
- “Groceries dropped after week 2.”

This turns Savvy from logger to analyst.

---

## 4. Budget Goals

Allow users to define soft budget targets.

Examples:
- Eating Out budget: ₹4,000
- Transport budget: ₹2,500

Savvy can then warn users proactively.

---

## 5. Smart Nudges

Savvy should become more proactive.

Examples:
- “You’re close to your Eating Out budget.”
- “Transport spend is unusually high this week.”
- “You’ve spent 70% of your flexible budget.”

This shifts Savvy from reactive tool to active copilot.

---

## 6. Recurring Expense Detection

Savvy should detect repeated transactions:
- subscriptions
- rent
- salary
- commute patterns

This improves prediction and planning.

---

## 7. Weekly / Monthly Summaries

Savvy should generate automatic summaries:
- weekly spend review
- monthly category breakdown
- overspend alerts
- savings suggestions

This increases passive value.

---

## 8. Safe-to-Spend Intelligence

Current safe-to-spend is rule-based.

V2 should improve this using:
- fixed expenses
- expected recurring spend
- savings targets
- remaining days in month

This makes spend guidance more trustworthy.

---

## 9. Natural Follow-Up Memory

Savvy should retain short conversational context.

Example:
User: “Coke Zero”
Savvy: “Should I log this under Eating Out, Groceries, or Personal?”
User: “Personal”

Savvy should resolve the original transaction without restarting.

This is critical for conversational UX maturity.

---

## 10. Financial Behavior Layer

Long term, Savvy should evolve from:
expense logger

to:
behavioral money assistant

The end-state is not better expense tracking.

The end-state is better spending behavior.
